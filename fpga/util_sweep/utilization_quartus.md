# scope_top utilization — Quartus (Agilex 3 A3CY100BM16AE7S)

Standalone `scope_top`, `RLE_EN=1` (STORE_W = PROBE_W+1), `XPORT="UART"`, full synth+fit.
Generated by `quartus_sweep.sh`.

| PROBE_W | DEPTH_LOG2 | STORE_W | ALMs | M20K | buffer bits (DEPTH×STORE_W) |
|---|---|---|---|---|---|
| 32 | 8  | 33 | 1,746 / 34,000 (5%) | 6 / 262 (2%)  | 8,448 |
| 32 | 12 | 33 | 1,819 / 34,000 (5%) | 13 / 262 (5%) | 135,168 |
| 32 | 15 | 33 | 1,964 / 34,000 (6%) | 69 / 262 (26%) | 1,081,344 |

Observations:
- **ALMs are ~flat across depth** (1.7k–2.0k): the logic is the trigger comparators + FSMs + CSR +
  RLE + drain, none of which scale with DEPTH — only the capture RAM does.
- **M20K scales with the buffer** (6 → 13 → 69), dominated by the DEPTH×STORE_W capture RAM plus
  the fixed cost of the 256×(DEPTH_LOG2+1) window-meta RAM and the two cmd/rsp async CDC FIFOs.
- This standalone build carries `XPORT="UART"` (its transport FIFOs cost a couple of M20K); the
  **#12 board build uses `XPORT="CSR"`**, so its `u_scope` capture path is 9 M20K at 32×12
  (8 = 4096×33 buffer + 1 = window-meta) — no transport FIFOs. See `../axc3000/README.md`.

The PROBE_W axis (128, 512) and the Vivado / Yosys+nextpnr flows are scripted in `quartus_sweep.sh`
+ `README.md` but not run in this environment (Vivado/nextpnr not installed; 512×2¹⁵ = 16.7 Mbit
exceeds this part's 262 M20K, as issue #12 anticipates — note it, don't skip).
